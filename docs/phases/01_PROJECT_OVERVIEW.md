# 🤖 WA-Platform - منصة بوتات الواتساب المتكاملة

## نظرة عامة

منصة **SaaS** احترافية لإدارة بوتات الواتساب تستهدف الشركات والأفراد في الشرق الأوسط.
تم تصميمها بعد دراسة معمّقة لمنصة **CAF Bot** (https://w.it-caf.com) ومعرفة نقاط ضعفها.

---

## 🎯 الفكرة الأساسية

المنصة تتيح لأي شخص أو شركة:
1. ربط رقم واتساب بالمنصة (عبر QR Code أو رقم الهاتف)
2. إرسال رسائل جماعية لآلاف العملاء
3. إنشاء ردود تلقائية ذكية
4. بناء بوت محادثة متكامل بدون برمجة
5. ربط الذكاء الاصطناعي للرد على استفسارات العملاء
6. تتبع كل شيء عبر تقارير مفصلة

## 🛠️ التقنيات المستخدمة

### Backend:
- **Node.js + Express.js** - السيرفر الرئيسي
- **PostgreSQL** - قاعدة البيانات
- **Redis** - Cache + Queue + Sessions
- **Bull.js** - إدارة مهام الإرسال الجماعي
- **Baileys** - مكتبة واتساب (غير رسمية)
- **Socket.IO** - تحديثات فورية (QR Code، حالة الرسائل)
- **JWT** - المصادقة
- **Multer + MinIO/S3** - رفع وتخزين الملفات
- **node-cron** - جدولة المهام

### Frontend:
- **React.js + Vite** - الواجهة الأمامية
- **Tailwind CSS** - التنسيق
- **Zustand** - إدارة الحالة
- **React Query (TanStack)** - جلب وكاش البيانات
- **React Flow** - Chat Flow Builder
- **Chart.js / Recharts** - الرسوم البيانية
- **i18next** - ترجمة عربي/إنجليزي
- **Socket.IO Client** - الاتصال الفوري

### الذكاء الاصطناعي:
- **Google Gemini 2.0 Flash** (الأرخص والأقوى)
- تكلفة ~$0.04 لكل 1000 رسالة

### Infrastructure:
- **Ubuntu 22.04 VPS**
- **Nginx** - Reverse Proxy
- **PM2** - إدارة العمليات
- **Cloudflare** - CDN + حماية DDoS
- **Let's Encrypt** - SSL مجاني
- **Docker Compose** (اختياري)

## 🔧 كيف تعمل المنصة (Architecture)

```
┌─────────────────────────────────────────────┐
│                   المستخدم                    │
│         (Browser / Mobile App)               │
└──────────────────┬──────────────────────────┘
                   │ HTTPS
┌──────────────────▼──────────────────────────┐
│                  Nginx                       │
│          (Reverse Proxy + SSL)               │
└──────────┬──────────────┬───────────────────┘
           │              │
     ┌─────▼─────┐  ┌─────▼──────┐
     │  Frontend │  │  Backend   │
     │  React.js │  │  Node.js   │
     │  (Static) │  │  Express   │
     └───────────┘  └─────┬──────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    ┌─────▼─────┐  ┌──────▼──────┐  ┌────▼────┐
    │PostgreSQL │  │    Redis    │  │ Baileys │
    │  (Main DB)│  │(Queue+Cache)│  │  (WA)   │
    └───────────┘  └─────────────┘  └────┬────┘
                                         │
                                   ┌─────▼──────┐
                                   │  WhatsApp  │
                                   │  Servers   │
                                   └────────────┘
```

### تدفق الرسالة الواردة:
```
عميل يرسل رسالة على واتساب
         ↓
Baileys تستقبل الرسالة
         ↓
Backend يفحص قواعد الرد التلقائي
         ↓
إذا وُجد تطابق → رد فوري
إذا لم يوجد + AI مفعّل → Gemini API → رد ذكي
         ↓
حفظ الرسالة في PostgreSQL
         ↓
إرسال للـ Webhook الخاص بالمستخدم (إن وُجد)
         ↓
تحديث لوحة التحكم عبر Socket.IO
```

### تدفق الإرسال الجماعي:
```
المستخدم ينشئ حملة (1000 رقم)
         ↓
Backend يُضيف 1000 مهمة لـ Bull Queue
         ↓
Workers يعالجون بتأخير عشوائي (3-7 ثواني)
لتجنب حظر واتساب
         ↓
تحديث حالة كل رسالة في الـ DB
         ↓
إشعار فوري للمستخدم عبر Socket.IO
```
